# 模型架构设计说明

## 概述

本文件说明影视制作智能体系统的模型架构、配置位置、以及模型选择决策机制。

---

## 一、系统架构概览

### 1.1 架构层次

```
┌─────────────────────────────────────────┐
│         用户交互层 (OpenClaw UI)         │
├─────────────────────────────────────────┤
│      工作区配置层 (Workspace Config)     │
│  SOUL.md | AGENTS.md | USER.md | 技能文件│
├─────────────────────────────────────────┤
│       Agent智能层 (AI Processing)         │
│  制片人 | 导演 | 服化道 | 分镜师         │
├─────────────────────────────────────────┤
│        模型调用层 (Model Router)          │
│  智能路由 | 任务分配 | 上下文管理        │
├─────────────────────────────────────────┤
│         底层模型 (LLM Backend)           │
│  OpenAI | Claude | 其他大模型           │
└─────────────────────────────────────────┘
```

### 1.2 数据流向

```
用户输入 → 工作区加载 → Agent识别 → 技能调用 → 
模型选择 → 任务处理 → 输出生成 → 持久化保存
```

## 三、模型选择决策机制

### 3.1 决策流程图

```
用户请求
    ↓
加载工作区配置 (SOUL.md + AGENTS.md)
    ↓
识别任务类型
    ↓
┌─────────────────────┐
│  根据任务选择模型  │
└─────────────────────┘
    ↓
┌─────────────────────────────────┐
│ 类型1：剧本分析/导演类  → 深思考模型 │
│ 类型2：服化道设计       → 创意模型   │
│ 类型3：分镜生成       → 详细模型   │
│ 类型4：审核/合规     → 严谨模型   │
└─────────────────────────────────┘
    ↓
调用相应技能配置
    ↓
执行任务
    ↓
保存到输出目录
```

### 3.2 任务类型到模型映射

| 任务阶段 | 推荐模型特性 | 优先级 | 考虑因素 |
|---------|------------|--------|--------|
| **剧本分析** | 深度思考、理解能力强、长文本 | 1. 创造性深度<br>2. 长文本处理<br>3. 分析能力 |
| **服化道设计** | 艺术创意、视觉描述能力 | 1. 艺术理解<br>2. 细节描述<br>3. 文化知识 |
| **分镜生成** | 格式规范、细致描述 | 1. 格式严谨<br>2. 镜头语言<br>3. 输出质量 |
| **审核任务** | 逻辑严谨、合规意识强 | 1. 逻辑严谨性<br>2. 规则遵循<br>3. 批评能力 |

## 四、具体模型配置说明

### 4.1 模型选择配置点

**配置位置**：OpenClaw系统设置（非本工作区管理）

**工作区与系统关系**：
- 本工作区负责提供：**任务上下文、能力描述、输出格式规范**
- OpenClaw系统负责：**模型实际选择、API调用、资源管理**

### 4.2 Agent特定模型偏好

在 `AGENTS.md` 中定义：

```markdown
# Agent配置

## 导演Agent
模型偏好：深度思考型
要求：剧本分析能力强，能理解复杂叙事结构

## 服化道设计师Agent
模型偏好：艺术创意型
要求：视觉描述能力强，有丰富文化知识

## 分镜师Agent
模型偏好：详细精确型
要求：Seedance 2.0格式规范，输出细致
```

### 4.3 技能级模型提示

在各 `SKILL.md` 文件中隐含：

- 技能的复杂程度 → 提示模型选择
- 输出格式的规范程度 → 影响模型选择
- 任务的创造性要求 → 影响模型选择

## 五、工作区加载流程

### 5.1 初始化加载顺序

```
1. 进入 workspace-video-producer/ 目录
2. OpenClaw 读取：
   a) 最先：.openclaw/workspace-state.json
   b) 然后：SOUL.md（系统核心身份）
   c) 然后：AGENTS.md（Agent角色）
   d) 然后：USER.md（用户偏好）
   e) 最后：BOOTSTRAP.md（启动引导）
3. 加载技能索引：skills/SKILLS_INDEX.md
4. 检查项目状态：project/.agent-state.json
5. 准备就绪
```

### 5.2 上下文构建过程

每次对话时：

```
1. 加载SOUL身份 → 确定"我是谁"
2. 加载AGENT配置 → 确定"我能做什么"
3. 加载USER偏好 → 确定"用户要什么"
4. 检查项目状态 → 确定"现在在做什么"
5. 构建完整上下文 → 发送给模型
6. 模型输出 → 按工作区规范执行
```

## 九、扩展与定制

### 9.1 添加新模型偏好

**方法**：编辑对应Agent或技能文件

**示例**：在 `AGENTS.md` 中添加
```markdown
## 导演Agent
模型偏好建议：使用XXX类型模型，擅长深度分析
```

### 9.2 工作区独立复制

- 整个 `workspace-video-producer/` 可复制
- 所有配置自包含
- 不依赖外部资源
- 可直接在其他OpenClaw实例中使用

## 十、总结

### 10.1 核心要点

1. **配置分层管理**：工作区 vs 系统级
2. **工作区自包含**：所有配置都在目录内
3. **模型选择提示**：通过能力描述间接提示
4. **状态持久化**：支持断点续传与学习积累
5. **灵活可扩展**：易于添加新能力与技能

### 10.2 关键文件速查

| 目的 | 文件位置 |
|-----|---------|
| 系统身份 | `SOUL.md` |
| Agent定义 | `AGENTS.md` |
| 技能索引 | `skills/SKILLS_INDEX.md` |
| 状态跟踪 | `project/.agent-state.json` |
| 用户偏好 | `USER.md` |
