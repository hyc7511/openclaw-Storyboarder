## 智能初始化机制

### 自动检查

当你开始工作时，系统会自动：
1. 检查 `project/evolution_log.db` 是否存在
2. 如果不存在，**自动创建**数据库
3. 如果已存在，**安全连接**并验证表结构

### 安全更新策略

- ✅ **只创建缺失的表**，不会覆盖已有数据
- ✅ **自动创建缺失的索引**，不影响现有数据
- ✅ **INSERT OR IGNORE**，安全初始化，零风险

## 问题反馈机制

### 简单反馈（推荐）

直接说问题，系统会自动处理：
```
用户：s003镜头角色越轴了！
系统：好的，已识别问题：角色越轴，责任方：分镜师
      已在后续迭代中增加：180度准则检查
```

### 详细反馈（可选）

提供更多信息帮助系统：
```
用户：问题描述：s003镜头角色越轴
      问题出现环节：分镜设计
      建议改进方向：检查180度准则
```

## 数据库管理（命令行，备用）

如果需要命令行管理，还是可以使用：

### 初始化
```bash
cd project
python init_db.py
```

### 查看状态
```bash
python evolution_db.py state
```

### 查看统计
```bash
python evolution_db.py summary
```

### 查看检查点
```bash
python evolution_db.py checkpoints
```

### 查看问题
```bash
python evolution_db.py list-issues
```

### 导出备份
```bash
python evolution_db.py export
```

### 完整帮助
```bash
python evolution_db.py help
```

## 从旧JSON迁移（如需要）

如果还有旧的JSON数据文件：

```bash
cd project
python evolution_db.py migrate .agent-state.json evolution_log.json
```

## 文件结构说明

### 静态配置（.md，Git管控）

```

├── SOUL.md              # 核心配置

```
project/
├── evolution_log.db     # SQLite数据库（主文件）
├── [临时文件...]        # SQLite日志文件
├── script/              # 剧本文件
├── assets/              # 素材库

```

└── init_db.py          # 初始化脚本
```
